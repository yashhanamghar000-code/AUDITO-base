AUDITO-AIThis repository contains the refactored, production-ready backend structure for the AUDITO-AI project. This architecture was collaboratively developed by Yash Hanamghar and Harsh Paygude
We have converted the original prototype into a coherent, scalable, and runnable structure. No new business logic was invented—the auth flow, upload flow, chat flow, hybrid search, and reranking are maintained as originally specified, now fully integrated into a FastAPI-based service.
Project Structure Highlightslegacy_prototypes/code-mm.py: Your original single-user, CLI-driven, Chroma-based prototype. It is not imported by the app; it is preserved purely as a reference for the original entity-routing and Chroma logic.API Additions: We added a DELETE /api/session/{session_id} endpoint in main.py to utilize the existing MultiUserRetriever.clear_session() method, enabling a "clear chat" functionality for your UI.
Running the ApplicationEnsure you have your environment variables configured before launching:Bashcd AUDITO-AI
cp backend/.env.example backend/.env   # Fill in MY_API_KEY, JWT_SECRET_KEY, etc.
docker compose up --build
